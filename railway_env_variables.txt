================================================================
NOTES:
================================================================

1. Replace <your-*> values with actual credentials
2. ${{Service.VARIABLE}} syntax automatically references other services
3. Generate APP_KEY for store-platform after first deployment:
   railway run --service store-platform php artisan key:generate --show
4. For security, use strong passwords (16+ characters)
5. Store sensitive keys in Railway's encrypted variables

================================================================
AFTER ADDING VARIABLES:
================================================================

1. Save variables
2. Redeploy the service (Railway auto-redeploys on variable changes)
3. Check deployment logs for any errors
4. Test the service endpoint
